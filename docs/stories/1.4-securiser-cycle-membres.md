# Story 1.4 — Sécuriser le cycle des membres

Statut : `user-validation`

## Objectif

Rendre atomiques les changements de rôle, retraits et départs, y compris sous concurrence.

## Critères d’acceptation

- [x] Seul un admin peut changer le rôle ou retirer un autre membre.
- [x] Tout membre peut quitter son groupe si un autre admin reste présent lorsque nécessaire.
- [x] Deux opérations concurrentes ne peuvent pas supprimer le dernier admin.
- [x] Les écritures directes équivalentes via REST sont refusées.

## Données protégées

`public.workspace_members` et l’existence d’au moins un administrateur par groupe.

## Rollback

Chaque RPC est transactionnelle. Une erreur annule intégralement l’opération.

## Validation

- Revue technique : RPC, matrice SQL et test concurrent validés sur local et restauration.
- Validation utilisateur : Pause 1.
