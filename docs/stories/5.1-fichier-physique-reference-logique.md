# Story 5.1 — Séparer fichier physique et référence logique

Statut : `user-validation`

## Objectif

Introduire une référence physique unique par objet R2 sans déplacer, réécrire ni supprimer les fichiers historiques.

## Contexte

Les assets actuels sont stockés dans `public.song_assets.storage_path`. Le manifeste R2 historique est disponible dans `docs/reports/R2_MANIFEST_FADERZERO_2026-07-20.csv`.

## Critères d’acceptation

- [x] `audio_files` contient une référence physique pour chaque `song_assets.storage_path` historique.
- [x] Les clés R2 existantes restent strictement inchangées.
- [x] La référence de `song_assets` est ajoutée sans casser `storage_path` ni les clients existants.
- [x] Toute divergence de taille, ETag ou référence manquante est consignée sans suppression.
- [x] Les comptages des assets ne diminuent pas pendant le backfill.

## Données protégées

- `public.song_assets`
- futurs `public.audio_files`
- objets du bucket R2 `faderzero-audio`
- manifeste R2 historique et objets orphelins

## Tâches

- [x] Comparer la migration avec le manifeste R2, sans opération d'écriture R2.
- [x] Exécuter les tests d'intégrité et documenter avant/après.
