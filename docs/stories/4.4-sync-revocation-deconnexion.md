# Story 4.4 — Sécuriser sync, révocation et déconnexion

Statut : `user-validation`

## Objectif

Empêcher tout push après révocation et toute déconnexion qui rendrait des mutations locales non sécurisées.

## Critères d’acceptation

- [x] Les appartenances sont relues auprès de Supabase avant chaque cycle de push en ligne.
- [x] Un workspace absent de la réponse serveur est purgé avant le push, cache audio inclus.
- [x] Les mutations, conflits et checkpoints du workspace révoqué sont supprimés localement.
- [x] La déconnexion tente de synchroniser tous les workspaces accessibles en écriture.
- [x] Hors ligne ou en échec, la déconnexion est bloquée tant qu’une mutation ou un conflit reste non sécurisé.
- [x] Sans mutation à risque, la déconnexion hors ligne reste possible car la base par utilisateur demeure isolée.
- [ ] Retrait réel d’un groupe puis reconnexion validé par l’utilisateur.

## Tests

- Purge d’un workspace révoqué : contenu autorisé conservé, contenu et queue révoqués supprimés.
- Déconnexion hors ligne avec mutation : refus et session conservée.
- Déconnexion hors ligne sans mutation : réussite.
- Suite complète : 115/115 tests, build réussi.

## Rollback

La purge ne touche que la base locale du compte et son cache partitionné. Les données serveur et la base historique restent intactes. Une réattribution prouvée peut être recopiée lors d’une connexion ultérieure.
