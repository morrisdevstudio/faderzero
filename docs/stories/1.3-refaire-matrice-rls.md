# Story 1.3 — Refaire la matrice RLS

Statut : `user-validation`

## Objectif

Appliquer côté base et côté Worker audio la matrice admin, membre, invité et non-membre.

## Critères d’acceptation

- [x] Admin : administration du groupe, membres, invitations et contenus.
- [x] Membre : lecture et écriture des contenus.
- [x] Invité : lecture, écoute et cache uniquement.
- [x] Non-membre : aucun accès.
- [x] `anon` ne possède plus de privilèges directs sur les tables métier.

## Données protégées

Toutes les tables métier et tous les objets du bucket R2 existant.

## Rollback

La migration précédente reste disponible ; aucune donnée n’est transformée par la matrice RLS.

## Validation

- Revue technique : matrice SQL locale/restaurée et 7 tests Worker validés.
- Correctif de recette : le rôle du workspace est chargé dans l’état client ; les actions d’écriture sont masquées ou bloquées pour `guest`, y compris la synchronisation montante.
- Preuves : requête REST locale `guest`, 70 tests Vitest, typecheck, lint et build.
- Validation utilisateur : Pause 1.
