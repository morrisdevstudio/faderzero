# Story 5.5 — Aligner la conversion locale

Statut : `user-validation`

## Objectif

Produire le même MP3 192 kb/s pour chaque nouvel upload sans exposer le nom visible dans la clé physique.

## Critères d’acceptation

- [x] WAV, M4A, FLAC et MP3 passent tous par le décodage et le réencodage local.
- [x] Le débit cible est fixé à 192 kb/s et le plafond de sortie à 50 Mo.
- [x] L’encodeur est importé statiquement en ESM dans un Web Worker module.
- [x] Le thread principal reçoit une progression et reste disponible pendant l’encodage.
- [x] Le nom visible est normalisé séparément ; la clé R2 reste un UUID.
- [x] Un échec de conversion intervient avant toute réservation ou écriture R2.
