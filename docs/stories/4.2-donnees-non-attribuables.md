# Story 4.2 — Traiter les données non attribuables

Statut : `user-validation`

## Objectif

Conserver les données `default-workspace` sans les attribuer automatiquement à un groupe non prouvé.

## Critères d’acceptation

- [x] Seules les appartenances renvoyées par Supabase ou déjà mises en cache hors ligne autorisent une copie.
- [x] `default-workspace` et les lignes sans workspace vont dans `recoveryItems`.
- [x] Les payloads, identifiants et relations sont conservés.
- [x] L’utilisateur peut rattacher les éléments à Mon espace depuis la page Sync.
- [x] Le rattachement crée des mutations `create` sans supprimer les données sources.
- [ ] Parcours réel de récupération validé par l’utilisateur.

## Tests

- Rattachement d’une chanson ambiguë : workspace personnel appliqué, statut `pending`, mutation créée.
- Réexécution : aucun doublon dans la zone de récupération.

## Rollback

Les entrées récupérées restent journalisées et leur payload d’origine demeure dans `recoveryItems`; la base historique reste disponible.
