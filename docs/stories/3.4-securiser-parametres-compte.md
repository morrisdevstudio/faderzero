# Story 3.4 — Sécuriser les paramètres du compte

Statut : `user-validation`

## Objectif

Sécuriser les changements d’e-mail et de mot de passe, terminer le parcours de récupération et révoquer toutes les sessions après un changement de mot de passe.

## Critères d’acceptation

- [ ] Le changement d’e-mail demande une nouvelle adresse et s’appuie sur la double confirmation Supabase.
- [ ] Le changement de mot de passe exige le mot de passe courant.
- [ ] Le nouveau mot de passe respecte les règles communes et sa confirmation correspond.
- [ ] Toutes les sessions sont révoquées après changement ou récupération du mot de passe.
- [ ] Le lien de récupération expire après une heure et ouvre un formulaire dédié.
- [ ] Aucun mot de passe n’est stocké dans le profil ou le stockage local.

## Données protégées

Identité Auth, sessions, profils, espaces, appartenances et contenus. Les opérations ne changent aucun identifiant utilisateur.

## Tests

- Services Auth : double confirmation, vérification du mot de passe courant et `signOut(global)`.
- Interface Compte : validations, messages et parcours recovery.
- Suite complète, typecheck, lint et build.

## Rollback

Masquer les formulaires sensibles sans modifier les comptes. Ne jamais restaurer un ancien mot de passe ni réactiver une session révoquée.
