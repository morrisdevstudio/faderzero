# Story 2.4 — Sécuriser le parcours sans compte

Statut : `done`

## Objectif

Conserver temporairement une invitation pendant la connexion ou l’inscription, sans laisser son secret dans l’URL, puis la revalider atomiquement au moment de l’adhésion.

## Critères d’acceptation

- [ ] Le paramètre `invite` disparaît immédiatement de l’URL.
- [ ] Le contexte survit au parcours de confirmation d’e-mail sur le même appareil.
- [ ] Le contexte local expire au plus tard après 24 heures.
- [ ] Les anciennes valeurs locales brutes sont migrées sans casser le parcours.
- [ ] L’invitation est relue puis consommée côté serveur après authentification.
- [ ] Les liens inconnus, expirés, révoqués ou consommés produisent un message neutre.

## Tests

- Arrivée sans session puis connexion.
- Inscription avec confirmation d’e-mail puis retour.
- Suppression immédiate du secret dans l’URL.
- Expiration du contexte local.
- Lien indisponible sans indication de sa cause.

## Rollback

Conserver les RPC sécurisées et revenir temporairement à l’ancien stockage local du contexte, sans réintroduire le token dans l’URL.

## Preuves

- 4 tests ciblés du contexte d’invitation réussis.
- Le test confirme le retrait immédiat du token de l’URL, l’expiration à 24 heures et la migration de l’ancienne valeur locale.
- Validation visuelle automatique indisponible : accès navigateur intégré à `127.0.0.1:5173` refusé par sa politique de sécurité.
