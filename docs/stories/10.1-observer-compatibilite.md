# Story 10.1 — Observer deux versions et trente jours

Statut : `in-progress`

## Objectif

Prouver durablement que les clients actifs ont terminé leur migration locale et qu’aucun ancien client ne dépend encore du schéma de compatibilité avant toute contraction.

## Garde-fou temporel

La première release instrumentée est `release-20260723-122850`, déployée le 23 juillet 2026. La date théorique la plus proche est le 22 août 2026, uniquement si une seconde version distincte a été observée et si toutes les métriques sont alors à zéro.

Ce délai ne peut être accéléré ni remplacé par un test ponctuel.

## Critères d’acceptation

- [x] Chaque build de déploiement embarque un identifiant de release distinct.
- [x] Un client authentifié remonte sa version, le schéma Dexie et le résultat de migration sans contenu utilisateur ni secret.
- [x] Le RPC refuse les appels anonymes et les valeurs invalides.
- [x] Un audit en lecture seule compte les clients, versions et éléments legacy sans afficher de token.
- [x] Le validateur échoue fermé lorsqu’une preuve manque.
- [ ] Deux versions compatibles distinctes ont été observées.
- [ ] Trente jours complets se sont écoulés depuis le début de l’observation.
- [ ] Tous les utilisateurs actifs ont au moins une observation récente.
- [ ] Aucun client ne nécessite de récupération locale.
- [ ] Aucun élément de récupération ou quarantaine ne reste non résolu.

## Données observées

La table privée conserve uniquement : utilisateur, UUID client aléatoire, version applicative, version Dexie, état de migration et compteurs. Elle ne conserve ni e-mail, ni token, ni contenu, ni nom d’appareil, ni empreinte matérielle.
