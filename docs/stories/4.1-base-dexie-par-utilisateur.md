# Story 4.1 — Créer une base Dexie par utilisateur

Statut : `user-validation`

## Objectif

Isoler les données locales de chaque compte sans supprimer ni modifier la base historique `faderzero-pwa`.

## Contexte

- [Plan global](../PLAN_IMPLEMENTATION_GLOBAL_FADERZERO.md)
- [Audit sécurité](../AUDIT_SECURITE_FADERZERO.md)
- La base historique v8 était globale et attribuait les anciennes lignes à `default-workspace`.

## Critères d’acceptation

- [x] Une base `faderzero-pwa-user-<userId>` distincte est activée avant l’affichage des données.
- [x] La copie est idempotente, table par table et reprenable après interruption.
- [x] Le journal conserve les comptages source/copie et les tables terminées.
- [x] Les lignes d’un workspace non autorisé ne sont pas copiées dans le compte courant.
- [x] La base historique n’est jamais supprimée.
- [ ] Installation réelle par-dessus la version historique validée par l’utilisateur.

## Données protégées

`songs`, `setlists`, `setlistSongs`, `songAssets`, `syncQueue`, `syncConflicts`, `syncState` et la base IndexedDB historique.
